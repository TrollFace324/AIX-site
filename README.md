# AIX - Команда по робототехнике

Сайт команды AIX с адаптивной вёрсткой, каруселями и переключаемым блоком робота: `2D` (изображение) / `3D` (GLB-модель).

## Возможности

- Адаптивный интерфейс для desktop/tablet/mobile.
- Мобильное меню и навигация по якорям.
- Карусель фото и карусель участников.
- Блок робота с переключением `3D` / `2D`.
- Ленивая загрузка 3D-модели (инициализация только после клика на `3D`).
- Управление 3D-моделью: смещение, вращение, зум, HUD (координаты/zoom).

## Технологии

- HTML5
- CSS3 (модульная структура файлов)
- Vanilla JavaScript (ES modules)
- Three.js (`three`, `GLTFLoader`, `DRACOLoader`, `OrbitControls`)

## Быстрый старт

### 1) Установить зависимости

```bash
npm install
```

### 2) Запустить локальный сервер

Любой статический сервер. Например:

```bash
# Python 3
python -m http.server 8000

# Или Node (http-server)
npx http-server
```

### 3) Открыть в браузере

`http://localhost:8000`

> Важно: 3D-режим не должен запускаться через `file://`. Используйте именно локальный сервер.

## Как работает блок робота

- По умолчанию показывается 2D-изображение: `images/images-png/robot-new.png`.
- В правом верхнем углу блока — кнопка `3D`.
- После нажатия `3D` загружается модель (`.glb`), затем кнопка меняется на `2D`.
- При нажатии `2D` возвращается изображение.

Порядок попытки загрузки моделей:

1. `3d-models/robot.glb`
2. `3d-models/robot-2.glb`
3. `3d-models/robot-1.glb` (если файл есть)

## Настройка 3D-поведения

Основные параметры находятся в `scripts/robot-viewer.js`, объект `VIEWER_SETTINGS`:

- стартовые параметры камеры;
- ограничения зума;
- частота рендера;
- шаг/частота смещения и поворота;
- ось поворота (`x` или `y`);
- ограничение смещения относительно центра модели.

## Публикация на GitHub Pages

Если планируете GitHub Pages:

- проверьте, что `node_modules` и `3d-models` доступны в деплое;
- либо перенесите необходимые runtime-файлы `three` в репозиторий (если хостинг не отдаёт `node_modules`).
